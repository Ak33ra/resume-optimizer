# Contact & Links

> **How to use:** copy this file to `CONTACT.md` (drop `.example`) and fill it
> in. The agent uses this for the resume header. Only include links you're happy
> to show on a resume. Different roles value different links — GitHub for SWE,
> Google Scholar for research, a portfolio site for startups.
>
> `CONTACT.md` is gitignored (it's PII). See `README.md` for the file set.

- **Personal site / portfolio:** (optional)
- **Google Scholar:** (research roles)
- **Other:** (X/Twitter, blog, Kaggle, Codeforces profile — include if it's a real signal)

"F-1, needs sponsorship after OPT". Not always on the resume, but useful context
for which roles to target.
